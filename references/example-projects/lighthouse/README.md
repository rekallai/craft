# Lighthouse — Example CRAFT Project

This is a **fictional** worked example of a CRAFT project. It exists to show what a fully populated tree looks like — every directory has at least one realistic file so you can read the standard in action without imagining what should go where.

**Lighthouse** is a made-up invoice-management SaaS for small-business owners who hate chasing payments. None of it is real. Treat the files here as illustrative templates, not as the truth about any actual product.

## Why this lives in `references/`

`references/` is for inputs an agent can read but shouldn't author against. An example project is exactly that — material to learn from, not a starting point you'd accidentally inherit when scaffolding your own work. Empty placeholders live at the top level of the CRAFT scaffold; the worked example lives here so a `git clone` gives you a clean tree.

## What to look at

- `specs/vision.md`, `specs/domain.md` — slow-moving human signal
- `specs/product/mvp-scope.md` — scoped product definition
- `specs/marketing/value-proposition.md` — positioning
- `references/google-meets/2026-04-08-customer-interview.md` — meeting transcript shape
- `references/slack/product/2026-04-12.md` — Slack dump shape
- `prompts/skills/code-review.md` — reusable skill
- `prompts/jobs/daily-pr-summary.md` — scheduled job with a trigger
- `prompts/qa/mvp-golden-path-acceptance-test.md` — QA routine
- `plans/engineering/mvp-data-model.md` — engineering plan with TLDR
- `plans/marketing-website/implementation-plan.md` — marketing plan
- `plans/style.css` — shared stylesheet for HTML plans
- `build/lighthouse-app/README.md` — placeholder pointing to a hypothetical shippable repo
